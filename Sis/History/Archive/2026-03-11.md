## Interaction 4: Zero-Trust Combat Engine v2 — Full Implementation

### What Was Discussed

This was a multi-session effort spanning the prior session (GameState + TurnMenu) and this session (ActionResolver tests, EncounterRunner, TacticsAdapter, and full integration verification).

### Architecture Overview
The engine-v2 is built on 5 core modules following zero-trust principles:
1. **GameState** — Immutable state container (Object.freeze, structural sharing)
2. **TurnMenu** — Option generation + validation (only legal choices presented)
3. **ActionResolver** — Action execution (validated option + state → new state + result)
4. **EncounterRunner** — Encounter loop orchestrator (immutable round processing)
5. **TacticsAdapter** — Bridge from v1 AI tactics to v2 menu-validated choices

Implemented from scratch (not ported from v1) with:

### Technical Decisions
1. **Immutability via Object.freeze** — All state transitions produce new frozen objects
2. **Injected getDecision function** — EncounterRunner has no AI dependency, receives decisions via callback
3. **Structural sharing** — GameState uses `_skipClone=true` internally for efficient immutable updates
4. **Graceful degradation** — TacticsAdapter falls back to dodge if v1 decision can't be matched to menu
5. **Spell slot relaxation** — If requested slot level unavailable, adapter finds any available slot for that spell
6. **Round normalization** — Encounter starts at round 0, increments at top of loop (matching v1 behavior)

**Bug 4:** `evalEnemyPolymorph` and `evalSelfPolymorph` are condition-based (not scored). Enemy polymorph is tried first (line 963) and fires against high-HP non-humanoid enemies. Self-polymorph (line 964) fires only when HP < 60% or 2+ enemies in melee and enemy polymorph conditions aren't met.

- The polymorph state machine has asymmetric save/restore between enemy and self paths — enemy path is less thorough
- The TacticsAdapter dodge fallback silently masks action matching failures, making bugs hard to detect from combat logs alone
- The combination of missing logs + missing flying restore creates a scenario that LOOKS like "sheep survived 10 rounds" but is actually "sheep died early, dragon reverted silently, dragon grounded and can't fight"
