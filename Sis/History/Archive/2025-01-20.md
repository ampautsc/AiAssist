# Sis History - January 20, 2025

## Session: Monarch Butterfly Movement Fix (Continued)

### Problem
Butterflies still weren't moving after multiple fix attempts. Some were floating off to infinity.

### Root Cause Discovery
Compared butterfly entity with vanilla Mojang entities (bee, bat, parrot) from bedrock-samples repo:

**Critical findings:**
1. **Missing `minecraft:can_fly` component** - ALL vanilla flying mobs have this
2. **Wrong component combination** - Was using parrot-style (`navigation.fly` + `movement.fly` + `behavior.random_fly`) which requires more complex setup
3. **Missing `minecraft:jump.static`** - Prevents jump behavior conflicts

### Solution Applied
Changed butterfly to use **bee-style hover pattern**:
- Added `minecraft:can_fly: {}`
- Added `minecraft:jump.static: {}`
- Changed `navigation.fly` → `navigation.hover`
- Changed `movement.fly` → `movement.hover`
- Changed `behavior.random_fly` → `behavior.random_hover` with `hover_height: [1, 4]`

### Files Modified
- `monarch_butterfly.json` - Complete rewrite with bee-style components
- `manifest.json` (BP) - Version bumped to 1.0.18
- `manifest.json` (RP) - Version bumped to 1.0.18

### Packs Rebuilt
- `monarch_garden_bp.mcpack` (8,424 bytes)
- `monarch_garden_rp.mcpack` (7,148 bytes)
- `monarch_garden.mcaddon` (12,773 bytes) - Contains both packs for single import

### Key Learning
Flying mob patterns in Bedrock:
- **BEE (hovering):** `navigation.hover` + `movement.hover` + `behavior.random_hover`
- **PARROT (flying):** `navigation.fly` + `movement.fly` + `behavior.random_fly`
- **BAT (floating):** `navigation.float` + `movement.basic` + `behavior.float_wander`

ALL require `minecraft:can_fly` component.

### Status
Ready for testing. Version 1.0.18 with correct bee-style hover behavior.

# Sis History - January 21, 2025

## Session: Major Butterfly Behavior Overhaul (v1.0.20)

### Checkpoint Created
`checkpoint_v1.0.19_20260121_205201.zip` - Backup before major changes

### Issues Fixed

#### 1. Invisible Milkweed Mounts (Purple/Black Checkerboard)
**Problem:** flower_perch was using `geometry.spider.v1.8` with missing texture
**Solution:** 
- Created `invisible.render_controllers.json` with `part_visibility: [{"*": false}]`
- Created `invisible.geo.json` with empty bones array
- Created 1x1 transparent PNG texture
- Updated `flower_perch.entity.json` to use invisible render controller

#### 2. Butterflies Bouncing Off Mounts
**Problem:** Butterfly had no mounting behavior
**Solution:**
- Set collision_box to 0 on flower_perch
- Added `pull_in_entities: true` to rideable component
- Reduced seat position to 0.2 height

#### 3. Wings Flapping During Glide
**Problem:** Animation controller used move_speed threshold but gliding still triggered flapping
**Solution:**
- Added new `animation.monarch_butterfly.glide` - wings extended at 45°, gentle oscillation
- Added new `animation.monarch_butterfly.roost` - wings closed at 5°
- Updated animation controller with 4 states: hovering, gliding, idle, roosting
- Glide triggers at `ground_speed > 0.2`, flutter at lower speeds

#### 4. Night Roosting Behavior
**Implementation:**
- Added `environment_sensor` with `is_daytime` filter
- Created `daytime_active` component group (hover + fly behaviors)
- Created `nighttime_roosting` component group (move_to_block targeting leaves)
- Events: `start_roosting` at sunset, `wake_up` at sunrise
- Butterflies seek oak/birch/spruce/jungle/acacia/dark_oak/azalea/mangrove/cherry leaves

### Files Created
- `render_controllers/invisible.render_controllers.json`
- `models/entity/invisible.geo.json`
- `textures/entity/invisible.png` (1x1 transparent)

### Files Modified

- `entities/flower_perch.json` - Zero collision, pull_in_entities
- `entity/monarch_butterfly.entity.json` - Added glide/roost animations
- `entity/flower_perch.entity.json` - Invisible rendering
