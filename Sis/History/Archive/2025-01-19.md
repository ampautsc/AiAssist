## Session: Deep Research - Tent Invisible and No Collision

### What Was Discussed
- Creator could place tent, could zip/unzip (saw door logs), but:
  - Tent was completely invisible
  - No collision/hitbox to bump into
- Needed deep research into entity rendering and solid entities

### Research Findings

**From wiki.bedrock.dev/entities/entity-intro-rp.html:**
- Client entity needs: materials, textures, geometry, render_controllers
- Render controllers use shortnames like `Geometry.default`, `Texture.default`, `Material.default`
- Capitalization matters! `Geometry.default` not `geometry.default`

**From wiki.bedrock.dev/entities/render-controllers.html:**
- `controller.render.default` is vanilla and expects `default` shortnames
- Custom render controller is more reliable - defines exactly what shortnames to use

**From wiki.bedrock.dev/entities/solid-entities.html:**
- **KEY FINDING:** `minecraft:is_collidable` component is what makes entities solid!
- Without this, entities have no physical collision with players/mobs
- Also learned about runtime identifiers (boat, shulker) for specific collision shapes

**From MS Learn damage_sensor docs:**
- `deals_damage` can be boolean OR string ("yes", "no", "no_but_side_effects_apply")
- Error message said "Allowed types: 'string'" - newer versions want STRING format
- Examples: `"deals_damage": "yes"` or `"deals_damage": "no"`

### Root Causes Identified
1. **Invisible tent**: Using `controller.render.default` which MAY work but custom is safer
2. **No collision**: Missing `minecraft:is_collidable` component!
3. **damage_sensor errors**: Using boolean `true`/`false` instead of string `"yes"`/`"no"`
4. **Old pack cached**: World was still running MonarchGar(1) - an old cached version

### Fixes Applied

**popup_tent.json (behavior):**
- Added `"minecraft:is_collidable": {}` component
- Changed `"deals_damage": true` → `"deals_damage": "yes"`

**torch_sensor.json (behavior):**
- Changed `"deals_damage": false` → `"deals_damage": "no"`

**popup_tent.entity.json (client):**
- Changed `controller.render.default` → `controller.render.monarch.popup_tent` (custom)

**popup_tent.geo.json:**
- Increased visible_bounds_width: 4 → 6
- Increased visible_bounds_height: 4 → 5
- Adjusted visible_bounds_offset for better centering

**New file created:**
- `render_controllers/popup_tent.render_controllers.json` - custom render controller

**manifest.json (both packs):**
- Version bumped from 1.0.25 → 1.0.26 so Minecraft recognizes as NEW pack

### Key Lessons Learned

1. **`minecraft:is_collidable`** - This is the magic component for solid entities! Without it, entities pass right through players.

2. **Custom render controllers** - More reliable than vanilla `controller.render.default`. Define exactly what shortnames you expect.

3. **damage_sensor string format** - Use `"yes"`, `"no"`, `"no_but_side_effects_apply"` NOT booleans

4. **Version bumping** - When fixing packs, bump version so Minecraft recognizes changes. Otherwise it may use cached old version.

5. **visible_bounds** - These control when entity renders at distance. If too small, entity disappears when player looks away.
