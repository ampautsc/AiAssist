#### Key Technical Insights
- Options P/L is piecewise linear - changes slope only at strike prices

- Price being optional is smart - you can still see the shape of the P/L curve

## Afternoon Session: Landscape Torch Feature for Monarch Garden

### What Creator Requested
A new motion-activated landscape torch for the Minecraft Monarch Garden addon with these features:
- Emits light **only downward** (not upward)
- **Amber colored** light
- **Motion activated** - turns on when entities are nearby
- Detection range matches light range (14 blocks)
- Shape: solid rimmed "hat" on top, conical V-shape glowing part underneath going down to the post
- Only the underside glows
- Normal torch brightness (level 14)

### What I Created

#### Files Created:
1. **Block Definition** - `behavior-packs/monarch_garden/blocks/landscape_torch.json`
   - Custom block with `monarch:lit` state (true/false)
   - Uses custom component `monarch:motion_sensor`
   - Light emission 14 when lit, 0 when off
   - Custom geometry reference
   - Separate material instances for hat, post, and glowing cone

2. **Motion Sensor Script** - `scripts/landscape_torch.js`
   - Efficient polling system (checks every 10 ticks / 0.5 seconds)
   - Detects players and mobs within 14-block radius
   - 3-second linger time before turning off
   - Registry-based tracking for all placed torches
   - Exported registration function integrated into main.js

3. **Custom Geometry Model** - `resource-packs/monarch_garden/models/blocks/landscape_torch.geo.json`
   - Post (bottom 6 pixels high)
   - Stepped conical section (layers from 6-10 pixels, narrowing upward)
   - Flat "hat" rim on top (8x8 pixels, 2 thick)
   - Material groups for different textures on each part

4. **Textures** (generated via Python script):
   - `landscape_torch_hat.png` - Dark slate/metal top
   - `landscape_torch_post.png` - Wooden brown with grain
   - `landscape_torch_amber.png` - Bright glowing amber when lit
   - `landscape_torch_off.png` - Dark amber when off

5. **Supporting Files**:
   - Updated `terrain_texture.json` with new texture references
   - Updated `blocks.json` with block texture mapping
   - Updated `en_US.lang` with localization
   - Created `loot_tables/blocks/landscape_torch.json`
   - Updated `main.js` to import and register the motion sensor component
   - Bumped version to 1.0.20 in both manifests

### Technical Notes
- The light physically still emits in all directions (Minecraft limitation), but the model shape creates the visual effect of downward lighting
- Motion detection uses efficient interval polling rather than per-tick checking
- Entity filtering includes players and common mobs, excludes items and projectiles
- 3-second linger prevents flickering when player is moving around

## Version 1.0.24 - Major Redesign

After testing, Creator provided a hand-drawn reference showing the correct design. I completely rebuilt the torch.

### Issues Found in Testing:
1. **Material mixing error** - Can't mix opaque/blend/alpha_test materials
2. **Entire block glowing** - Bone-based materials don't work for blocks in Bedrock
3. **Motion always triggered** - Player always in range, bad logic
4. **Night detection broken** - Logic error (always true)

### Solution - Texture Atlas Approach:
Instead of multiple materials per bone, I created a **single 32x32 texture atlas** with UV mapping:
- Post wood texture in one region
- Hat metal in another region  
- Hat BOTTOM (amber glow) in its own region
- Each face of the geometry maps to the correct UV region

### New Model (per Creator's sketch):
- Simple **T-shape**: flat disc hat on narrow post
- **Post**: 2x8x2 pixels (narrow wooden stem)
- **Hat**: 10x2x10 pixels (wide flat disc)
- Only the **underside of the hat** glows amber when lit!

### Fixed Motion Detection:
- Now uses **entity count tracking** to detect actual movement
- Only activates at night (12500-23500 ticks) or underground (y<50)
- 5-second linger time
- 10-block detection radius

### Files Changed:
- `landscape_torch.geo.json` - Simplified T-shape with proper per-face UV
- `landscape_torch_off.png` & `landscape_torch_lit.png` - 32x32 atlases
- `landscape_torch.json` - Single material, simpler config
- `landscape_torch.js` - Fixed light/motion logic

## Update: Event Triggers Master List Created

Creator wanted to augment the verified-patterns library with a comprehensive list of ALL event triggers. "Events are like gold" - they avoid polling!

### What I Created

**EVENT_TRIGGERS_MASTER_LIST.md** - A comprehensive reference of every way to trigger actions in Minecraft Bedrock without polling:

#### 1. Script API World Events (afterEvents) - 45+ events
- Player events: join, leave, spawn, dimension change, emote, hotbar change
- Block events: break, place, interact, explode
- Entity events: spawn, die, hurt, health change, load, remove
- Combat events: hit entity, hit block, projectile hits
- Redstone events: button push, lever action, pressure plates, tripwire, piston
- World events: weather change, world load, game rule change

#### 2. Script API World Events (beforeEvents) - 10 cancelable events
- Can cancel: block break, item use, explosion, effect add, interactions

#### 3. System Events
- scriptEventReceive: Bridge between /scriptevent commands and scripts

#### 4. Block Custom Component Events - 11 methods
- onPlayerInteract, onStepOn, onStepOff, onEntityFallOn
- onPlace, onBreak, onPlayerBreak
- onRandomTick, onTick, onRedstoneUpdate
- beforeOnPlayerPlace

#### 5. Item Custom Component Events - 7 methods
- onUse, onUseOn, onHitEntity, onMineBlock
- onConsume, onCompleteUse, onBeforeDurabilityDamage

#### 6. Entity Sensor Components (Data-Driven JSON)
- entity_sensor: Detects entities within range
- environment_sensor: Monitors environment conditions continuously
- damage_sensor: Triggers on taking damage
- timer: One-shot or repeating delays
- scheduler: Time-of-day based scheduling
- interact: Player interaction events

#### 7. Entity Filter Tests - 70+ conditions
- State tests: is_daytime, is_sleeping, on_fire, in_water...
- Property tests: has_tag, has_component, has_target...
- Environment tests: is_brightness, is_biome, weather, moon_phase...
- Relationship tests: is_family, is_owner, distance_to_nearest_player...

#### 8. Animation Controller Events
- on_entry and on_exit for state transitions

### Also Created: event-patterns.js

- "I want to detect when X happens" → Here's the event to use
