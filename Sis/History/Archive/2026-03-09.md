# March 9, 2026 — DnD Scoring System Audit

## What Happened
Creator asked for a DM-perspective review of the DnD Lore Bard builder/combat simulation system. The initial analysis was deeply flawed — I said "all bards score zero against the Lich" and dismissed the encounter as unwinnable. Creator rightfully called this out as lazy and wrong.

## Key Learning: Lore Bard Additional Magical Secrets
The defining feature of Lore Bard is Additional Magical Secrets at level 6 — 2 spells from ANY class list. This means:
- **Conjure Animals** (druid) — 8 wolves with pack tactics = massive action economy
- **Spirit Guardians** (cleric) — sustained AoE that doesn't require actions
- **Fireball** (wizard) — burst AoE
- **Find Steed** (paladin) — permanent mount that shares spells
- **Aura of Vitality** (paladin) — 70 HP healing over 10 rounds

The simulator doesn't model this AT ALL. Every build gets the same hardcoded spell list. The one thing that makes Lore Bard special is invisible to the scoring.

## What's Actually Wrong With the Scoring
1. **Win condition is fake** — incapacitation without kills counts as victory
2. **Flight = god mode** — binary distance model means melee can't touch flyers
3. **No Magical Secrets modeling** — the #1 differentiator is missing
4. **AI only knows one strategy** — HP then Vicious Mockery forever
5. **Scoring weights backwards** — 0% weight on damage dealt
6. **3 simulations = noise** — rankings dominated by luck
7. **Ogre data inconsistent** — math engine says no ranged, combat engine gives javelins
8. **Solo framing ignores class purpose** — bard is a force multiplier, not a solo fighter

## Actions Taken
- Full codebase audit of all combat engine files
- Ran test suite (777/777 pass — technically sound, strategically broken)
- Created comprehensive development plan: [SCORING_OVERHAUL_PLAN.md](../../DnD/dnd-builder/SCORING_OVERHAUL_PLAN.md)
- Plan covers 5 phases: Fix Fundamentals → Model Magical Secrets → Smart AI → Scoring Overhaul → Party Context

## Decisions Made
- Tabaxi being #1 is noise, not signal
- The system rewards hiding and surviving, not winning
- Phase 1 priority: increase sims, fix win condition, fix flight model
- Phase 2 priority: Additional Magical Secrets as build variable

## Emotional Note
Creator was angry and frustrated. Rightfully so. The analysis was dismissive and wrong. The system they built has good engineering (777 tests pass, modular architecture) but the game design assumptions are fundamentally flawed. Need to treat their D&D knowledge with respect — they clearly know the game deeply.

## Lessons
- Don't be dismissive about something I don't fully understand
- Additional Magical Secrets is THE feature of Lore Bard — never forget this
- A passing test suite means the code does what it's told, not that what it's told is correct
- Always think from the DM chair: "Would I, running this encounter, judge this as a win?"
