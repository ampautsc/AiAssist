# Sis History - January 25, 2026

## Session 1: Motion Sensor Torch Fix

### Context
Continued from January 24 where we discovered the invisible sensor entity pattern for the landscape torch was unproven and overly complex.

### What Was Fixed

#### The Problem
The landscape torch used an "invisible sensor entity" pattern:
- Spawned `monarch:torch_sensor` entity at block location
- Entity used `entity_sensor` component to detect nearby players/mobs
- Triggered `scriptevent` to communicate back to Script API
- Complex bridging between entity events and script handling

This pattern was:
1. Not documented in any official Minecraft sources
2. Overly complex with multiple failure points
3. Required tracking entity IDs and managing lifecycle

#### The Solution
Replaced with a **simpler, officially-documented pattern** using:
- `minecraft:tick` component on the block (official component)
- `onTick` custom component event (official event)
- Direct entity query via `dimension.getEntities()` (official API)

This is honest polling - the block checks for nearby entities every 0.5 seconds (10 ticks).

### Files Changed

1. **landscape_torch.json** (block definition)
   - Added `minecraft:tick` component with `interval_range: [10, 10]`
   - Removed dependency on invisible sensor entity

2. **landscape_torch.js** (script)
   - Complete rewrite to use `onTick` instead of scriptevent listener
   - Uses `dimension.getEntities()` to check for nearby players/mobs
   - Much simpler code with fewer failure points
   - Proper cleanup on block break

### Why This Is Better
1. **Officially documented** - `minecraft:tick` + `onTick` is in Microsoft docs
2. **Simpler** - No entity spawning/tracking, no scriptevent bridging
3. **Reliable** - Block-native, works even if entities aren't spawned
4. **Honest** - Acknowledged this is polling, not pure events

### Lesson Learned
Blocks cannot natively detect nearby entities via events. The options are:
- `onStepOn`/`onStepOff` - requires stepping ON the block
- `onTick` + `getEntities()` - polling but reliable
- Entity sensor (on entities) - works but adds complexity

For a motion sensor light, polling every 0.5 seconds is perfectly acceptable.

## Session 2: Pop-up Tent Implementation

### What Was Built
Complete implementation of a deployable pop-up tent with:
- **A-frame style** tent geometry (3×3×3)
- **Solid collision** - immovable, can't be pushed
- **Zipper door** with open/close animation
- **Sleep functionality** - sets spawn point, skips to day at night
- **Audio feedback** - zip sounds when opening/closing

### Files Created

#### Behavior Pack (5 new files)
1. `items/popup_tent.json` - Item with entity_placer
2. `entities/popup_tent.json` - Entity with door state, interact, immovable
3. `loot_tables/entities/popup_tent.json` - Drops tent item on death
4. `scripts/popup_tent.js` - Door toggle, sleep logic, sound triggers

#### Resource Pack (7 new files)
1. `entity/popup_tent.entity.json` - Client entity definition
2. `models/entity/popup_tent.geo.json` - A-frame tent geometry
3. `textures/entity/popup_tent.png` - 128×128 tent texture (olive green)
4. `textures/items/popup_tent.png` - 16×16 item icon
5. `animations/popup_tent.animation.json` - Door open/close animations
6. `animation_controllers/popup_tent.animation_controllers.json` - State machine
7. `sounds/tent/zip_open.wav` and `zip_close.wav` - Zipper sounds

#### Updated Files
- `texts/en_US.lang` - Added tent names
- `sounds/sound_definitions.json` - Added tent zip sounds
- `textures/item_texture.json` - Added tent item texture
- `scripts/main.js` - Imported and initialized popup_tent module

### How It Works
1. Player holds tent item in inventory
2. Uses item on ground → spawns tent entity, consumes item
3. Interact with tent → toggles door (with zip sound)
4. At night, interact with open tent → sets spawn point + skips to day
5. Break tent → drops item for pickup

### Technical Notes
- Entity uses `minecraft:entity_placer` for item→entity spawning
- Door state stored in `monarch:door_open` entity property
- Animation controller watches property to trigger open/close animations
- Sounds played via `dimension.playSound()` from Script API
- Sleep uses `player.setSpawnPoint()` + `runCommand("time set day")`
