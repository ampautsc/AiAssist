## Interaction: Seed Dispersal Mechanics Refinement

### What was discussed
Creator proposed changing seed dispersal from scripted rise/hover/fall physics to simpler bee-like floating behavior. Seeds would float slowly until it rains, then gravity turns on and they fall quickly to plant themselves.

### What was learned
- **Simplicity wins:** Native hover movement components are better than custom scripted physics
- **Rain as trigger:** `world.getDimension().getWeather()` can detect rain/thunder
- **Component groups for state switching:** Can swap between floating (gravity off) and falling (gravity on) via events
- **Bee movement pattern:** Uses `movement.hover`, `navigation.hover`, and `behavior.random_fly` - perfect template for slow floating seeds

### Decisions made
Updated SEED_DISPERSAL_PLAN.md to reflect new approach:
- Seeds spawn from mature milkweed
- Float slowly using hover movement (speed 0.08 vs butterflies 0.3-0.5)
- Rain detection triggers event that enables gravity
- Seeds fall quickly and plant on dirt/grass/farmland
- Invalid surfaces = despawn after 2 seconds
- 2 minute max lifetime (vs previous 30 seconds)

### Actions taken

1. Updated "Seed Entity Behavior" description

4. Added component groups for floating vs falling states

8. Reorganized implementation phases (now 6 phases including rain detection)

### Technical insights
The new approach:
- **Uses proven patterns:** Same components butterflies use
- **Simpler code:** No complex physics calculations
- **Better gameplay:** Players see seeds floating, creates atmosphere
- **Natural trigger:** Rain makes ecological sense for seed dispersal
- **Cleaner state machine:** Two clear states (floating/falling) instead of three phases (rise/hover/fall)

### Emotional response
Satisfied with the elegant simplification. The rain trigger is much more intuitive than arbitrary timer-based physics phases. This feels like how nature actually works - seeds drift until moisture brings them down.

## Interaction: Auto-Collection Instead of Punching

### What was discussed
Creator questioned why players need to punch seeds - why not auto-collect like normal items?

### What was learned
Entities (needed for floating movement) don't auto-collect like dropped items by default. But we can script proximity detection to mimic auto-collection behavior.

### Decisions made
Changed from punch-to-collect to walk-over auto-collection:
- No need for health/loot/damage components
- Script checks player proximity (1.5 block range)
- Automatically adds seed to inventory when player walks near
- Plays `random.pop` sound on collection
- Much better UX - matches Minecraft's normal item collection
